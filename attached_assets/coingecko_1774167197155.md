# CoinGecko API Reference

## Overview

CoinGecko is the canonical source for token prices, market cap, supply metrics, and metadata.
Free tier works without API key (or with a demo key for slightly better limits).

**Base URLs:**
- Free: `https://api.coingecko.com/api/v3`
- Pro: `https://pro-api.coingecko.com/api/v3` (header: `x-cg-pro-api-key`)

---

## ID Resolution — ALWAYS DO THIS FIRST

CoinGecko uses its own internal IDs, not ticker symbols. IDs are often unintuitive.

| Endpoint | Description |
|---|---|
| `GET /search?query={term}` | Search by name/symbol — returns list of matches with `id` |
| `GET /coins/list` | Full list of all coins with `id`, `symbol`, `name` |
| `GET /coins/list?include_platform=true` | Includes contract addresses per chain |

**Common gotchas:**
- `bitcoin` not `btc`, `ethereum` not `eth`, `solana` not `sol`
- `lido-dao` not `lido`, `uniswap` not `uni`
- `tether` not `usdt`, `usd-coin` not `usdc`
- Wrapped tokens have separate IDs: `wrapped-bitcoin`, `staked-ether`
- Multiple tokens may share a symbol. Always verify by `id`.

**Best practice:** Call `/search?query=` first, pick the correct `id`, then use it downstream.

## Core Endpoints

### Current Price

| Endpoint | Description | Key Params |
|---|---|---|
| `GET /simple/price` | Current price for multiple tokens | `ids` (comma-sep), `vs_currencies` (e.g., `usd`), `include_market_cap`, `include_24hr_vol`, `include_24hr_change` |

```
GET /simple/price?ids=bitcoin,ethereum&vs_currencies=usd&include_market_cap=true&include_24hr_change=true
```

Response:
```json
{
  "bitcoin": { "usd": 67000, "usd_market_cap": 1320000000000, "usd_24h_change": 2.5 },
  "ethereum": { "usd": 3500, "usd_market_cap": 420000000000, "usd_24h_change": 1.8 }
}
```

### Token by Contract Address

| Endpoint | Description |
|---|---|
| `GET /simple/token_price/{platform}` | Price by contract address |
| `GET /coins/{platform}/contract/{address}` | Full coin data by contract |

`platform` = `ethereum`, `polygon-pos`, `arbitrum-one`, `optimistic-ethereum`, `base`, `binance-smart-chain`, `solana`, `avalanche`

### Detailed Coin Data

| Endpoint | Description |
|---|---|
| `GET /coins/{id}` | Full metadata, market data, community data, dev stats |
| `GET /coins/{id}?localization=false&tickers=false&community_data=false&developer_data=false` | Lean version (faster) |

**Key fields in response:**
- `market_data.current_price.usd` — current price
- `market_data.market_cap.usd` — market cap
- `market_data.fully_diluted_valuation.usd` — FDV
- `market_data.circulating_supply` — circulating
- `market_data.total_supply` — total (including locked)
- `market_data.max_supply` — max (null if no cap)
- `market_data.total_volume.usd` — 24h trading volume
- `market_data.ath.usd`, `market_data.atl.usd` — ATH/ATL
- `categories` — token categories
- `platforms` — contract addresses per chain
- `description.en` — project description
- `links` — website, twitter, github, etc.

### Historical Prices

| Endpoint | Use | Granularity |
|---|---|---|
| `GET /coins/{id}/market_chart?vs_currency=usd&days={n}` | Price + volume + mcap over time | Auto: 5min (<1d), hourly (1-90d), daily (>90d) |
| `GET /coins/{id}/market_chart/range?vs_currency=usd&from={ts}&to={ts}` | Custom date range | Same auto-granularity |
| `GET /coins/{id}/ohlc?vs_currency=usd&days={n}` | OHLC candles | `days`: 1, 7, 14, 30, 90, 180, 365, max |
| `GET /coins/{id}/history?date={dd-mm-yyyy}` | Snapshot at specific date | Single data point |

**Granularity rules for `/market_chart`:**
- `days=1` → ~5 min intervals
- `days=2..90` → hourly
- `days=91+` or `max` → daily
- Pro plan unlocks `interval=5m` override for up to 30 days

**OHLC candle intervals:**
- 1-2 days → 30min candles
- 3-30 days → 4hr candles
- 31-365 days → daily candles

### Markets / Rankings

| Endpoint | Description | Key Params |
|---|---|---|
| `GET /coins/markets` | Top tokens by market cap | `vs_currency`, `order`, `per_page` (max 250), `page`, `category`, `ids` |

Useful for: building token screeners, getting bulk market data, filtering by category.

```
GET /coins/markets?vs_currency=usd&order=market_cap_desc&per_page=100&page=1&sparkline=false
```

### Categories

| Endpoint | Description |
|---|---|
| `GET /coins/categories/list` | All category names + IDs |
| `GET /coins/categories` | Categories with market cap + volume aggregates |

### Exchanges

| Endpoint | Description |
|---|---|
| `GET /exchanges` | All exchanges with volume, trust score |
| `GET /exchanges/{id}/tickers` | Trading pairs on an exchange |

## Rate Limit Strategy

**Free tier: ~30 requests/min** (enforced, will 429)

Strategies:
1. **Batch IDs:** `/simple/price?ids=btc,eth,sol,...` — up to ~100 IDs per call
2. **Use `/coins/markets`** for bulk data instead of individual `/coins/{id}` calls
3. **Cache `/coins/list`** — only changes when new tokens are listed
4. **Add `&precision=2`** to price calls to reduce response size (Pro only)
5. **Retry with backoff:** On 429, wait 60s before retrying

**Demo key:** Register at CoinGecko for a free demo API key — same rate limits but more stable.

## Common Patterns

### Get full token profile for a model
```
1. GET /search?query={name}               → Resolve ID
2. GET /coins/{id}                         → Full metadata + market data
3. GET /coins/{id}/market_chart?days=365   → 1-year price history
```

### Bulk token comparison
```
1. GET /coins/markets?ids={comma,list}&vs_currency=usd  → Side-by-side market data
```

### Price by contract address (when you don't know CoinGecko ID)
```
1. GET /simple/token_price/ethereum?contract_addresses={addr}&vs_currencies=usd
```
Or use DefiLlama's `coins.llama.fi/prices/current/ethereum:{addr}` which is often easier.

### Historical snapshot for a model
```
1. GET /coins/{id}/history?date=01-01-2025  → Market data at that date
```

## Known Limitations

- Free tier caps historical data at daily granularity beyond 90 days
- `/coins/{id}` is heavy (~50KB per call) — use lean params or `/simple/price` for just prices
- Exchange volume data includes wash trading; use `trust_score` to filter
- Some new/small tokens may not be listed yet — fall back to DefiLlama contract price
- Category assignments can be subjective/incomplete
- `max_supply` is null for many tokens (no hard cap defined)
